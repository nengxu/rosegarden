How to get Rosegarden

Coming soon: Buy it boxed!

Rosegarden will reach 1.0 this summer, and then we intend to make box sets available. (We're old-fashioned like that, and we've had quite a bit of interest in the prospect.)

The box will include Rosegarden binary packages for the major Linux distributions, as well as a bootable version that you can run direct from CD, plus plenty of other goodies including a number of other audio applications and plugins and a Windows VST adapter. With a printed manual and support included, this box set should be pretty exciting value.

What's more, because Rosegarden will always be Free Software licensed under the GNU General Public License, you can depend on the same friendly licensing as always and the right to enjoy your software without intrusive activation schemes or other annoyances. We see users as allies, not enemies.

Downloads

Note: please read the Hardware and System Requirements page to find out whether Rosegarden will work on your system.

Source code

  • Rosegarden release 0.9.6 sources, 12th January 2004, for download from the SourceForge page.
  • Rosegarden CVS from the SourceForge repository; this is the latest bleeding-edge, work-in-progress and might not build or run but will have all the latest features as discussed on the rosegarden-devel mailing list.

Binary packages

These are available are all maintained by other people, so it may take a short while before they are brought up-to-date with each new release of Rosegarden and there's no guarantee of their completeness. Please take a moment to check whether you are really getting the latest release (currently Rosegarden-0.9.6) before you download a binary package.

Rosegarden for Mandrake

  • Rosegarden for Mandrake, one of Thac's amazing list of audio and music application RPMs -- click through according to your version of Mandrake and scroll down to find Rosegarden.

Apparently there's also an RPM in the Mandrake Cooker bleeding-edge package repository.

Rosegarden for SuSE

We don't know of any source of up-to-date SuSE packages of Rosegarden. SuSE have included Rosegarden in their last two distributions, but neither of the packages worked very well.

We recommend building from source on SuSE: two of the three core Rosegarden developers use SuSE and you should be able to compile Rosegarden out of the box on SuSE 8.2 or 9.0.

Rosegarden for RedHat

  • Rosegarden for RedHat 7.3, 8.0, and 9.0 from Planet CCRMA.

    Planet CCRMA is a well-maintained and easy to use distribution of audio and sound applications for RedHat, which can install everything you need semi-automatically. If you want to set up an audio workstation on RedHat, your best bet is to install the Planet CCRMA system: read the documentation on the Planet site, and go from there.

If you want to run Rosegarden on RedHat but you don't want to use Planet CCRMA, you're probably on your own. It's a difficult distribution to run much modern audio software on, because it doesn't include the ALSA 0.9.x sound drivers.

Rosegarden for Debian

  • Rosegarden is available as part of Debian unstable, thanks to Enrique Robledo Arnuncio.

Rosegarden for your distribution

Several distributions are now including versions of Rosegarden, if not in the base distribution then in an extended package repository. For example, Mandrake, SuSE, Debian and the Russian language ALT Linux all make Rosegarden packages available to their users.

If we're not packaged as part of your favourite distribution then please get in touch with your distribution manufacturer and ask why not! If you've made an RPM or package for any distribution then please let us know so that we can link it here.


"X11 Rosegarden" 2.1 releases

Before the current Rosegarden project, there was another Rosegarden: a now rather antique program, simpler, less powerful but also less resource-hungry. We're calling it "X11 Rosegarden" here, to distinguish it from the newer Rosegarden. It was originally written for Silicon Graphics IRIX machines, and then ported to Linux and other Unix variants. If you're interested, you can still get it in source code form right here.

  • "X11 Rosegarden" 2.1 patchlevel 4 sources via http (573K).
  • "X11 Rosegarden" 2.1 patchlevel 3 sources via http (570K), and patch from 2.1 patchlevel 2 to patchlevel 3 (7K).
  • "X11 Rosegarden" 2.1 patchlevel 3 for Linux, binaries via http (379K). Built on a Debian 2.2 system.
  • Torsten Duwe's patch to make "X11 Rosegarden" 2.1 use the OSS /dev/sequencer2 API, via http only (24K). This patch also includes some of the fixes in 2.1pl2. It should be applied to the original 2.1 sources; it will not apply correctly to 2.1pl2 or pl3.
  • "X11 Rosegarden" 2.1 for SGI IRIX 6.2, binaries via ftp or via http (501K). This distribution lacks the Petal filter scripting language, because the SGI machine I have access to doesn't have new enough Tcl/TclMIDI components to build it.
  • "X11 Rosegarden" 2.1 for OpenVMS Vax or Alpha, binaries via http in the UK (3.6Mb) -- ported by Patrick Moreau.
  • "X11 Rosegarden" 2.1 sources via ftp or via http (553K).
